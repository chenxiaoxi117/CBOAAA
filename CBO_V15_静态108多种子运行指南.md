# CBO V15 静态108多种子运行指南

## 推荐策略

先用 `core` 口径运行 BO 与 adaptive CBO。这是验证 CBO 相对 BO 的核心数据，也是多种子必须重跑的部分。

```text
METHOD_SET=core
方法数=2
每个 seed：108个任务、216个 round summary
```

最终需要统一展示固定策略和贪心策略时，再对一个主 seed 使用 `full`：

```text
METHOD_SET=full
方法数=6
每个 seed：108个任务、648个 round summary
```

## 旧静态结果能否复用

BO 和 CBO 不能复用：V15 使用新参数范围、CBO-first reference、internal6、无 backlog growth penalty 和 V1.4 adaptive acquisition；每个随机种子都必须重新运行。

旧 fixed/direct 方法只有在模拟器版本、seed、场景、窗口240、总时长120000、task adaptation 和调度归一化配置完全一致时，实际延迟/能耗等指标才可复用。旧 normalized score 可能使用不同 reference，不建议直接拼接。

因此推荐：

1. seed43/44/45 均跑 `core`。
2. 需要最终六方法统一归一化主表时，seed43 再跑一次 `full`。
3. 如果论文要求六种方法的多种子置信区间，则所有 seed 都必须跑 `full`。

## 上传

```text
D:\CBOv2\run_static108_v15_adaptive.sh
    -> /home/ecs-user/CBO/run_static108_v15_adaptive.sh
```

服务器应使用最新 `new_tr_split`。

## 单个种子

```bash
tmux new -s cbo108_s43
cd /home/ecs-user/CBO
source env.sh
MAX_JOBS=7 METHOD_SET=core bash run_static108_v15_adaptive.sh 43
```

## 三个种子顺序运行

不要同时启动三个各7并发的脚本。一个 tmux 中顺序执行：

```bash
tmux new -s cbo108_multiseed
cd /home/ecs-user/CBO
source env.sh

for seed in 43 44 45; do
  MAX_JOBS=7 METHOD_SET=core bash run_static108_v15_adaptive.sh "$seed" || break
done
```
