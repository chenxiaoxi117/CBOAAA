# V1.3: Static Reference Baseline And BO/CBO Comparability

## 0. Current V1.3 Implementation Update

The local code now implements the requested CBO-first shared reference protocol.

```text
CBO source method runs first within the normal BO budget
-> the first CBO warm-up windows build the scenario normalization reference
-> BO and other methods reuse this CBO-derived reference
-> if the same scenario signature appears again, reuse the cached reference
-> if CBO sees a new scenario signature, warm up and cache a new reference
```

This does not add extra experiment rounds. If `--bo-iterations 500` is used,
the reference warm-up windows are part of those 500 windows.

Default V1.3 config:

```text
CBO_SHARED_REFERENCE_POLICY = cbo_first
CBO_SHARED_REFERENCE_WARMUP_ROUNDS = 5
CBO_REFERENCE_SOURCE_METHOD_KEY = reduced7_cbo_lite_pressure_taskmix_counts
```

CLI overrides:

```text
--cbo-shared-reference-policy cbo_first
--cbo-shared-reference-warmup-rounds 5
--cbo-reference-source-method-key reduced7_cbo_lite_pressure_taskmix_counts
```

Expected logs:

```text
[ScenarioReference] policy=cbo_first source=...
[SCENARIO-REF] method=... policy=cbo_first source_method=... is_source=...
[ScenarioReference] CBO-first cached reference signature=... source=...
```

The previous fixed-probe discussion below is retained as historical context and
as the alternative mode available through:

```text
--cbo-shared-reference-policy fixed_probe
```

## 1. This Check

This note clarifies the normalization-reference issue found in the
`window240_nogrowth_top5_static_bo_cbo_v5` rerun.

The main question was whether the code already supports the intended protocol:

```text
CBO runs first, builds a normalization reference, and BO then uses that CBO-derived reference.
```

Before the V1.3 code update, this exact CBO-first-to-BO protocol was not the active static experiment path.

## 2. Previous Fixed-Probe Path

The implemented shared-reference path is:

```text
fixed probe policy -> build frozen scenario reference -> BO/CBO/direct/fixed methods share it
```

This is implemented in:

```text
scenario_experiments.py
prepare_shared_scenario_normalization_reference()
```

The function chooses a fixed method through:

```text
_choose_reference_probe_group(groups)
```

Current priority:

```text
reduced6_fixed_mid
reduced4_fixed_mid
any method with fixed_theta
```

If no fixed method is present in `selected_keys`, the code prints:

```text
[WARN] no fixed probe policy found; scenario normalization reference will use per-method fallback
```

In that fallback case, BO and CBO each build/freeze their own online reference during calibration, so their normalized scores are not guaranteed to use the same denominator.

## 3. What V1.1/V1.2 Already Say

`V1_1.md` states that scenario normalization should be built by a fixed probe policy before the main experiment, and then shared by Fixed/BO/CBO/direct methods.

`V1_2.md` mainly tightens dynamic phase reference behavior. It introduces phase signatures and common reference banks for dynamic experiments, and notes that reference calibration windows should be separated from main post-calibration results.

Neither file describes a confirmed static protocol where CBO is run first and BO then reuses the CBO-derived reference.

## 4. What Happened In The BO+CBO v5 Rerun

The run used:

```text
selected_keys = reduced7_bo_greedy,reduced7_cbo_lite_pressure_taskmix_counts
BO_INTERVAL = 240
CBO_BACKLOG_GROWTH_PENALTY_WEIGHT = 0.0
```

Because no fixed method was included, the shared fixed-probe reference was not built.

Therefore:

```text
BO normalized score = BO result normalized by BO fallback reference
CBO normalized score = CBO result normalized by CBO fallback reference
```

This means normalized score is useful as a process diagnostic, but it is weaker as a strict cross-method headline metric.

Raw metrics are more reliable for this run:

```text
delay
unfinished_rate
backlog
violation_rate
energy_per_arrival
```

## 5. Current Robust Conclusion

For `window240_nogrowth_top5_static_bo_cbo_v5`, the structure is mostly consistent:

```text
same scenarios
same seed shown in stdout: seed=43
same BO_INTERVAL=240
same CBO backlog growth penalty disabled
same task arrival counts between BO and CBO
```

The robust finding is:

```text
CBO shows stronger middle/late-stage convergence and service-side stability in these difficult scenes.
```

Specifically, in the five difficult scenes:

```text
last50 normalized score: CBO better in 4/5
service_norm: CBO better in 4/5
unfinished_rate: CBO better in 4/5
backlog: CBO better in 4/5
last100 score stability: CBO better in 4/5
time to within 3% of final last50: CBO faster in 4/5
```

But because the reference is per-method fallback, the safest wording is:

```text
CBO appears better in service stability and convergence under the current run,
but a shared-reference rerun is needed before using normalized score as the main fairness metric.
```

## 6. Recommended V1.3 Static Protocol

For fair static BO/CBO comparison, use a shared fixed-probe reference.

Run the same scenes with:

```text
selected_keys = reduced7_fixed_mid,reduced7_bo_greedy,reduced7_cbo_lite_pressure_taskmix_counts
```

The fixed method is included to trigger shared reference construction. It does not have to be the final comparison target; it is needed to create one common scale.

Expected successful log:

```text
[ScenarioReference] built ... shared reference(s) with probe=reduced7_fixed_mid ...
```

Expected output files:

```text
scenario_normalization_reference.json
scenario_normalization_reference_cache.json
```

Expected no-warning condition:

```text
No "no fixed probe policy found" warning
```

Then compare BO and CBO using:

```text
normalized_tradeoff_score
service_norm
energy_norm
raw delay
raw unfinished_rate
raw backlog
raw violation_rate
raw energy_per_arrival
```

## 7. Alternative Protocol: CBO-Derived Reference

If the intended design is explicitly:

```text
CBO warm-up/reference run -> save reference bank -> BO and CBO load the same CBO-derived reference
```

then the experiment should be two-stage:

1. Run CBO-only reference calibration and export:

```text
--cbo-reference-output-file <reference_bank.json>
```

2. Rerun BO+CBO with:

```text
--cbo-reference-mode load
--cbo-reference-file <reference_bank.json>
```

This would make BO and CBO share the CBO-derived scale.

However, this is a different fairness choice from fixed-probe reference. It favors a CBO-defined notion of scene scale and should be named clearly in the paper as:

```text
CBO-derived shared reference
```

The fixed-probe protocol is more neutral for BO-vs-CBO headline comparison.
