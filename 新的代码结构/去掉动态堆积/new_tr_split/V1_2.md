# V1.2 动态阶段 Reference 修改说明

## 1. 修改目标

本版不改变 BO/CBO 主框架，只把动态实验的归一化 reference 规则收紧为：

```text
阶段触发式 reference
相似阶段复用 reference
显著新阶段使用前 K 个 warm-up 窗口建立 reference
阶段内 reference 冻结
所有方法共享同一套 reference_bank
主结果优先看 post-calibration 窗口
```

静态 scenario 的 reference 逻辑基本保持不变；主要修改对象是 `dynamic_scenario`。

## 2. 动态 Reference 规则

动态实验现在按 phase signature 管理 reference：

```text
phase_signature = lambda + task_mix + deadline_pressure
                + resource_perturbation_id
                + link_profile_id
                + task_adaptation_enabled
```

进入一个动态 phase 时：

```text
如果与已有 phase 相似：
  复用已有 active_reference_id
  不重新 warm-up

如果是显著新 phase：
  使用 K 个 reference warm-up 窗口建立该 phase 的 reference
  reference 写入 SCENARIO_NORMALIZATION_REFERENCE_CACHE
  phase 内固定不再滑动更新
```

默认 K：

```text
PHASE_REFERENCE_WARMUP_ROUNDS = 5
```

可通过命令行修改：

```bash
--phase-reference-warmup-rounds 5
```

## 3. Reference 由哪些 raw 指标构成

`_cbo_build_reference()` 中的 reference 现在包含：

```text
delay_ref
energy_per_arrival_ref
energy_norm_ref
unfinished_rate_ref
backlog_ref
backlog_growth_ref
backlog_growth_rate_ref
rt_violation_rate_ref
success_rate_ref
eval_cost_ref
arrivals_ref
```

统计方式仍由原参数控制：

```bash
--cbo-reference-stat median
--cbo-reference-trim-pct 0.1
```

默认推荐 `median`，避免 warm-up 中个别异常窗口把尺度拉偏。

## 4. Backlog 归一化

之前 backlog growth 作为 raw penalty 进入 `service_norm`。

V1.2 起，如果 reference 中存在有效的：

```text
backlog_growth_rate_ref
```

则优先使用：

```text
backlog_growth_rate_norm
```

进入 service penalty；如果 reference 不可用或为 0，则退回原来的 raw `backlog_growth_rate`。

这保证 backlog 也进入动态阶段尺度，而不是只对 delay/energy 做 reference。

## 5. Warm-up 标记

动态新 phase 的前 K 个窗口会被标记为：

```text
is_calibration_window = True
calibration_window_label = warm_up
```

相似 phase 复用旧 reference，不标记 warm-up。

新增或补齐的日志字段：

```text
phase_iter
phase_reference_warmup_rounds
phase_reference_is_new_scene
phase_reference_base_phase_id
reference_source
phase_reference_cache_status
backlog_ref
backlog_growth_ref
backlog_growth_rate_ref
backlog_norm
backlog_growth_norm
backlog_growth_rate_norm
```

## 6. 动态汇总文件

`dynamic_round_summary.csv` 现在会导出 reference 与 warm-up 字段。

`dynamic_phase_summary.csv` 新增 post-calibration 统计：

```text
phase_warmup_rows
phase_post_calibration_rows
phase_post_calibration_mean
phase_post_calibration_first20
phase_post_calibration_first50
phase_post_calibration_last50
```

论文主结果建议优先使用：

```text
phase_post_calibration_mean
phase_post_calibration_last50
```

而不是直接使用包含 warm-up 的 `phase_mean`。

## 7. 运行示例

动态场景：

```bash
python -m new_tr_split ^
  --mode dynamic_scenario ^
  --selected-keys reduced7-fixed-mid,reduced7-bo-greedy,reduced7-cbo ^
  --dynamic-schedule "1.0:30,40,30:150;1.05:32,38,30:150;2.0:10,10,80:200" ^
  --phase-reference-warmup-rounds 5 ^
  --cbo-objective-mode normalized_tradeoff ^
  --cbo-reference-mode calibrate ^
  --output-root D:\CBOv2\results\dynamic_v12
```

说明：

```text
phase 1 建 reference；
phase 2 如果被判定为相似，则复用 phase 1 reference；
phase 3 如果变化明显，则建立新 reference；
每个新 reference phase 的前 5 轮标记 warm_up。
```

## 8. 验证结果

已运行短动态 smoke test：

```bash
python -m new_tr_split ^
  --mode dynamic_scenario ^
  --selected-keys reduced7-fixed-mid ^
  --dynamic-schedule "1.0:30,40,30:3;1.05:32,38,30:3;2.0:10,10,80:3" ^
  --repeat 1 ^
  --phase-reference-warmup-rounds 2 ^
  --no-boltzmann-random ^
  --output-root D:\CBOv2\results\quick_dynamic_ref_v12
```

检查结果：

```text
reference cache 数量 = 2
phase 1 / phase 2 复用同一个 phase_signature
phase 3 使用新的 phase_signature
phase 1 前 2 轮 warm_up=True
phase 2 warm_up=False
phase 3 前 2 轮 warm_up=True
dynamic_phase_summary.csv 已生成 post-calibration 字段
```
