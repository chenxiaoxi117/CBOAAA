# new_tr_split 运行指导

本文档记录当前版本常用运行命令、主要参数、可选方法和实验口径。建议从 `D:\CBOv2\新的代码结构` 目录运行：

```powershell
Set-Location "D:\CBOv2\新的代码结构"
python -m new_tr_split --mode scenario --output-root "D:\CBOv2\results\scenario_test"
```

## 1. 当前默认口径

当前代码默认使用：

```text
CBO_OBJECTIVE_MODE = normalized_tradeoff
CBO_REFERENCE_MODE = calibrate
SCHEDULER_SCORE_NORM_MODE = candidate_minmax_deadline
USE_TASK_TYPE_ADAPTATION = true
```

含义：

- `normalized_tradeoff`：BO/CBO 训练目标使用归一化后的 service-energy tradeoff score，越小越好。
- `calibrate`：实验开始前先用固定 probe policy 校准 reference，再让各方法共享 reference。
- `candidate_minmax_deadline`：底层节点评分按当前任务的候选节点集合归一化 delay / energy / queue / risk。
- `task_node_affinity_factor` 默认打开，表示共享环境异构，不是 CBO 独有能力。

BO/CBO 方向已确认：

```text
外部 cost 越小越好
agent.tell(theta, cost) 内部记录 y = -cost
acquisition 最大化 y
```

所以 `normalized_tradeoff_score` 越小越好，与 BO 内部优化方向一致。

## 3. 选择方法

使用 `--selected-keys` 指定要跑的方法，逗号分隔：

```powershell
python -m new_tr_split `
  --mode scenario `
  --selected-keys "fixed_mid,bo-greedy,cbo-ptc,greedy-direct-cost" `
  --output-root "D:\CBOv2\results\selected_test"
```

常用方法别名：

```text
fixed_mid                         -> reduced6_fixed_mid
fixed_tuned                       -> reduced6_fixed_tuned
bo-greedy                         -> reduced6_bo_greedy
cbo-ptc                           -> reduced6_cbo_lite_pressure_taskmix_counts
cbo-pressure                      -> reduced6_cbo_lite_pressure_only
cbo-taskmix                       -> reduced6_cbo_lite_taskmix
cbo-full-taskmix-counts           -> reduced6_cbo_lite_full_taskmix_counts
alpha-direct                      -> reduced6_cbo_alpha_direct
alpha-direct-no-risk              -> reduced6_cbo_alpha_direct_no_risk
ad-prev-unfinished                -> reduced6_cbo_alpha_direct_prev_unfinished_context
greedy-direct-cost                -> direct_greedy_cost
least-load-direct                 -> direct_least_load
queue-aware-greedy-direct         -> direct_queue_aware_greedy
rr-direct                         -> direct_round_robin
```

方法含义：

- `fixed_mid`：固定中等权重 probe / baseline。
- `fixed_tuned`：固定调优基线。注意：当前目标改成 `normalized_tradeoff` 后，这个需要重新调参。
- `bo-greedy`：无上下文 BO，适合作为 BO no-context。
- `cbo-ptc`：CBO 使用 pressure + task mix + counts 上下文，是当前推荐 CBO 主方法。
- `alpha-direct`：用 alpha 重参数化 delay-energy tradeoff。
- `direct-*`：不经过 BO/CBO 的直接调度基线。

## 4. Context / Task Adaptation 消融

建议消融矩阵：

```text
BO no-context:
  bo-greedy

CBO task-context only:
  cbo-taskmix

CBO resource/pressure-context only:
  cbo-pressure

CBO full / mixed context:
  cbo-ptc
  或 cbo-full-taskmix-counts

with task adaptation:
  --task-adaptation

without task adaptation:
  --no-task-adaptation
```

示例：

```powershell
python -m new_tr_split `
  --mode scenario `
  --selected-keys "bo-greedy,cbo-taskmix,cbo-pressure,cbo-ptc" `
  --task-adaptation `
  --output-root "D:\CBOv2\results\ablation_adapt_on"
```

```powershell
python -m new_tr_split `
  --mode scenario `
  --selected-keys "bo-greedy,cbo-taskmix,cbo-pressure,cbo-ptc" `
  --no-task-adaptation `
  --output-root "D:\CBOv2\results\ablation_adapt_off"
```

解释口径：

- `--task-adaptation` 打开共享 `task_node_affinity_factor`。
- `--no-task-adaptation` 所有任务只看节点基础 `service_rate_gips`。
- 这不是 CBO 专属增强，所有方法共享同一环境设定。

## 5. 归一化目标和 reference

### 训练目标

```powershell
--cbo-objective-mode normalized_tradeoff
```

可选：

```text
normalized_tradeoff  主实验推荐，训练用归一化 tradeoff score
eval_cost            使用窗口原始 cost
diagnostic_only      只记录 normalized 指标，不作为训练目标
```

### reference 模式

```powershell
--cbo-reference-mode calibrate
--cbo-reference-calibration-rounds 30
--cbo-reference-stat median
```

可选：

```text
calibrate   实验前用 probe policy 校准并冻结 reference，推荐
off         不使用 reference，normalized_tradeoff 可能退回 eval cost
load        从 JSON 文件读取 reference
auto_macro  类似 calibrate，保留宏场景 key 兼容
```

动态实验中会输出：

```text
scenario_normalization_reference.json
scenario_normalization_reference_cache.json
```

日志中关注：

```text
phase_signature
active_reference_id
reference_source
is_calibration_window
normalized_tradeoff_score
bo_training_cost_source
```

### 跨目录比较必须统一 reference

如果要比较不同参数范围或不同主线，例如：

```text
reduced6
alpha_default
alpha_wide
alpha_mid
```

不能让每个实验目录各自校准 reference。正确做法是先生成一个 common reference bank，再让所有实验加载它。

第一步：生成 common reference bank。

```powershell
python -m new_tr_split `
  --mode pressure_scan `
  --lambda-values "1.0,1.8,2.6" `
  --task-probs "30,40,30" `
  --selected-keys "fixed_mid" `
  --bo-iterations 30 `
  --cbo-reference-mode calibrate `
  --cbo-reference-calibration-rounds 30 `
  --cbo-reference-output-file "D:\CBOv2\results\common_reference_bank_balanced.json" `
  --output-root "D:\CBOv2\results\build_common_reference_balanced"
```

第二步：所有要横向比较的实验都加载同一个 bank。

```powershell
python -m new_tr_split `
  --mode pressure_scan `
  --lambda-values "1.0,1.8,2.6" `
  --task-probs "30,40,30" `
  --selected-keys "bo-greedy,cbo-ptc" `
  --bo-iterations 500 `
  --cbo-reference-mode load `
  --cbo-reference-file "D:\CBOv2\results\common_reference_bank_balanced.json" `
  --output-root "D:\CBOv2\results\v12_reduced6_common_ref"
```

alpha default / wide / mid 也都使用同一个：

```powershell
--cbo-reference-mode load
--cbo-reference-file "D:\CBOv2\results\common_reference_bank_balanced.json"
```

这样才能比较跨目录的：

```text
normalized_tradeoff_score
service_norm
energy_norm
```

否则 normalized score 只能在同一个实验目录、同一个 lambda 内部比较。

当前代码会按静态场景签名选择 reference：

```text
lambda
task mix
deadline pressure
resource_perturbation_id
link_profile_id
task_adaptation_enabled
```

所以：

```text
lambda=1.0, 30/40/30 -> ref_A
lambda=1.8, 30/40/30 -> ref_B
lambda=2.6, 30/40/30 -> ref_C
```

同一个 common bank 里会保存多个签名。运行时会自动按当前 lambda / task mix 取对应 reference。

## 7. alpha 重参数化

启用 alpha 方案：

```powershell
--scheduler-tradeoff-mode alpha_direct
--selected-keys "alpha-direct"
```

含义：

```text
w_rt_delay    = alpha_rt
w_rt_energy   = 1 - alpha_rt
w_batch_delay = alpha_batch
w_batch_energy= 1 - alpha_batch
w_ai_delay    = alpha_ai
w_ai_energy   = 1 - alpha_ai
```

默认范围：

```text
alpha_rt:    0.70 - 0.98
alpha_batch: 0.50 - 0.90
alpha_ai:    0.50 - 0.95
```

命令行覆盖：

```powershell
--alpha-direct-rt-bounds "0.70,0.98"
--alpha-direct-batch-bounds "0.50,0.90"
--alpha-direct-ai-bounds "0.50,0.95"
```

queue / risk 范围仍来自 `core_config.py`：

```text
CONTROL_QUEUE_BOUNDS
CONTROL_RISK_SCALE_BOUNDS
```

建议调参时检查：

```text
top theta 是否大量贴边
risk 项是否长期支配 score
queue 项是否导致过度避开高容量节点
RT violation 是否真的下降
```

### 任务比例


```powershell
--task-prob-schedule "0:4000:30,40,30;4000:8000:10,10,80"
```

### RT deadline

```powershell
--rt-deadline-factor 2.5
```

值越小，RT 越紧；值越大，RT 越宽松。

### 云资源扰动

```powershell
--cloud-delay-mult 1.2
--cloud-energy-mult 1.1
--cloud-service-rate-mult 0.9
```

含义：

- `cloud-delay-mult > 1`：上云传输更慢。
- `cloud-energy-mult > 1`：上云传输更耗能。
- `cloud-service-rate-mult < 1`：云服务率降低。

如果这些扰动表示一个新外部场景，建议同步在 `core_config.py` 设置：

```text

PHASE_LINK_PROFILE_ID = "link_delay_high"
```

## 13. fixed-tuned 重调提醒

只要发生以下任一变化：

```text
训练目标从 eval_cost 改为 normalized_tradeoff
底层 score 归一化改变
参数范围改变
alpha 重参数化
task adaptation 默认打开或关闭
动态场景训练集改变
```

`fixed-tuned` 都必须重新调参，否则它代表旧目标下的强基线，和新 CBO 不公平。

建议重新调参流程：

```text
normalized_tradeoff
+ 当前 scheduler-score-norm-mode
+ 当前 task_adaptation 设置
+ 当前动态场景训练集
+ 当前 bounds
-> grid / random search
-> 锁定新的 fixed-tuned theta
-> 再进入正式测试
```

## 14. reduced7 / reduced9 能耗权重方案

这次新增两条介于 `reduced6` 和 `alpha_direct` 之间的路线，用来验证“能耗权重固定为 1.5 是否合理”。

### reduced7：全局能耗参数

控制向量：

```text
theta7 = [
  W_RT_Latency,
  W_Batch_Latency,
  W_AI_Latency,
  W_Queue,
  W_Risk_Scale,
  Cloud_Gate,
  W_Energy_Scale
]
```

映射关系：

```text
W_RT_Energy = W_Energy_Scale
W_Batch_Energy = W_Energy_Scale
W_AI_Energy = W_Energy_Scale
```

含义：只增加 1 维，让 BO/CBO 学一个全局节能倾向。它适合作为下一步候选主线，因为比 `alpha_direct` 更自由，又比任务级能耗参数更容易收敛。

常用别名：

```text
reduced7-fixed-mid
reduced7-fixed-tuned
reduced7-bo-greedy
reduced7-cbo / reduced7-cbo-ptc / cbo7
```

能耗权重范围可用命令行覆盖：

```powershell
--reduced7-energy-scale-bounds "0.5,3.0"
```

### reduced9：任务级能耗参数

控制向量：

```text
theta9 = [
  W_RT_Latency,
  W_Batch_Latency,
  W_AI_Latency,
  W_RT_Energy,
  W_Batch_Energy,
  W_AI_Energy,
  W_Queue,
  W_Risk_Scale,
  Cloud_Gate
]
```

含义：允许 RT / Batch / AI 三类任务有不同能耗敏感性。它更灵活，但维度从 6 增加到 9，500 轮内可能更难稳定收敛，所以建议作为增强消融，不要马上替代主线。

常用别名：

```text
reduced9-fixed-mid
reduced9-fixed-tuned
reduced9-bo-greedy
reduced9-cbo / reduced9-cbo-ptc / cbo9
```
