# CBO V14 留出12场景验证运行指南

## 目的

V13 Top5 已参与机制设计。本实验使用12个未参与调试的场景，分别覆盖旧 CBO 较差、接近持平和较好三类，验证 V1.4 自适应探索能否泛化。

本轮不改代码和参数，只换场景。每个场景运行：

```text
V14-A_diag: calibration 只诊断，纯预测均值
V14-B_soft: 固定 soft 探索
V14-C_adaptive: V1.4 自适应探索
```

每组同时包含 BO 与 CBO，共 `12 × 3 = 36` 个实验任务、72个 round summary。

## 上传

上传：

```text
D:\CBOv2\run_holdout12_v14_adaptive_exploration_s43.sh
    -> /home/ecs-user/CBO/run_holdout12_v14_adaptive_exploration_s43.sh
```

服务器应已经包含最新 `new_tr_split` 和 `analyze_top5_sigma_abc.py`。

## 运行

```bash
tmux new -s cbo_v14
cd /home/ecs-user/CBO
source env.sh
MAX_JOBS=5 bash run_holdout12_v14_adaptive_exploration_s43.sh 43
```

脱离 tmux：`Ctrl+B`，再按 `D`。

## 查看进度

```bash
cd /home/ecs-user/CBO
watch -n 30 '
echo "Completed: $(find result/holdout12_v14_adaptive_exploration_s43 -name pressure_scan_summary_all.csv -type f | wc -l) / 36"
echo "Running: $(pgrep -fc "python.*new_tr_split")"
'
```

## 完成标准

```text
pressure summaries = 36 / 36
refactor configs   = 36 / 36
round summaries    = 72 / 72
```

## 分析

```bash
cd /home/ecs-user/CBO
source env.sh
python analyze_top5_sigma_abc.py \
  result/holdout12_v14_adaptive_exploration_s43 \
  --variants V14-A_diag V14-B_soft V14-C_adaptive
```

重点看 adaptive 在三类场景中的胜率，而不仅是12场景平均值。若留出场景仍保持前期改善、后期不反弹，再进入多 seed 和108场景主实验。
